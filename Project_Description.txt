1. PROJECT OVERVIEW
----------------------------------------------------------

The Trust-Aware Federated Learning Based Intrusion Detection System (TAFL-IDS) is a distributed Artificial Intelligence framework developed to detect malicious network traffic in Internet of Things (IoT) environments while preserving user privacy.

Traditional Intrusion Detection Systems (IDS) rely on centralized training, where all client devices transmit their raw network traffic to a central server. Although effective, this approach introduces significant privacy, scalability, and security concerns, particularly in IoT ecosystems where devices generate sensitive and geographically distributed data.

To overcome these limitations, this project adopts Federated Learning (FL), allowing each client device to train a local deep learning model using its own network traffic without sharing raw data. Instead, only model parameters are communicated to a central aggregation server.

A Trust Management mechanism is incorporated into the federated learning process to evaluate the reliability of participating clients. Trust scores are dynamically assigned based on client performance and are used during model aggregation to reduce the influence of malicious or unreliable participants.

The result is a privacy-preserving, trust-aware, and intelligent intrusion detection framework capable of learning collaboratively while resisting compromised clients.


==========================================================
2. PROBLEM STATEMENT
----------------------------------------------------------

Conventional Intrusion Detection Systems suffer from multiple challenges:

• Dependence on centralized datasets
• Privacy leakage due to raw data transmission
• Vulnerability to malicious participants during collaborative learning
• Poor scalability across distributed IoT environments

Although Federated Learning addresses privacy concerns by keeping data local, standard federated aggregation assumes that every participating client behaves honestly. This assumption is unrealistic in practical IoT deployments where compromised devices may intentionally upload poisoned model updates.

This project addresses these limitations by introducing a Trust-Aware Aggregation mechanism that evaluates the reliability of each client and weights its contribution during global model generation.

4. SYSTEM ARCHITECTURE
----------------------------------------------------------

The system consists of the following components:

1. IoT Clients
   • Maintain local datasets
   • Train local CNN + BiLSTM models
   • Never transmit raw data

2. Local Deep Learning Model
   • CNN extracts spatial network traffic features
   • BiLSTM captures sequential temporal patterns
   • Generates updated model weights

3. Trust Manager
   • Evaluates client behaviour
   • Updates trust scores dynamically
   • Penalizes unreliable or malicious clients

4. Trust-Aware Aggregator
   • Collects local model weights
   • Performs weighted aggregation using trust scores
   • Generates improved global model

5. Global Model Repository
   • Stores the latest CNN + BiLSTM model
   • Redistributes updated model to all participating clients

6. Flask REST API
   • Serves project metrics
   • Executes federated simulations
   • Performs real-time intrusion prediction
   • Provides generated analytics graphs

7. React Dashboard
   • Displays live project statistics
   • Visualizes federated learning process
   • Shows client trust values
   • Displays training analytics
   • Performs intrusion prediction

6. FEDERATED LEARNING WORKFLOW
----------------------------------------------------------

Step 1:
Dataset is distributed among multiple simulated clients.

Step 2:
Each client trains its own CNN + BiLSTM model locally.

Step 3:
Training accuracy is evaluated.

Step 4:
Trust Manager updates trust scores.

Step 5:
Model parameters are transmitted to the server.

Step 6:
Trust-Aware Aggregator combines local weights according to client trust.

Step 7:
Updated global model is redistributed.

Step 8:
Process repeats for multiple communication rounds.


==========================================================
7. TRUST MANAGEMENT
----------------------------------------------------------

Each client maintains an independent trust score.

Trust values increase when:

• Local model performs well.
• Client behaves consistently.

Trust values decrease when:

• Accuracy deteriorates significantly.
• Client exhibits malicious behaviour.
• Poisoned labels are detected during simulation.

During aggregation, clients with higher trust contribute more strongly to the global model while low-trust clients receive reduced influence.

This improves robustness against model poisoning attacks.
